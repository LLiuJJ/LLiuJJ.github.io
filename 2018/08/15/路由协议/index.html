路由协议

Posted on 2018-08-15 |

路由控制的定义

  • 为了让数据包能够正确的发送到目标主机，路由器必须在途中进行正确地转发。这种向“正确的方向”转发数据所进行的处理就叫做路由控制或路由。
  • 静态路由的设置通常是实现设置好路由器和主机并将路由信息固定的一种方法。
  • 动态路由是指让路由协议在运行过程中自动地设置路由控制信息的一种方法。

路由控制范围

  • IGP - Interior Gateway Protocol (内部网关协议)

  • EGP - Exterior Gateway Protocol (外部网关协议)

  • IGP中还可以使用RIP(Routing Information Protocol，路由信息协议)、RIP2、OSPF(Open Shortest Path First, 开放最短路径优先)等众多协议。

  • EGP使用的是BGP(Porder Gateway Protocol，边界网关协议)协议

路由算法

距离向量算法

路由器之间可以呼唤网络的方向及其距离相关信息，并以这些信息制作路由控制表。这种方法在处理上比较简单，不过由于只有距离和方向的信息，所以当网络变得十分复杂的时候，在获得稳定路由信息之前需要消耗一定的时间。

链路状态算法

路由状态算法是路由器在了解网络整体连接状态的基础上生成路由控制表的一种方法。该方法中，每个路由器必须保持同样的信息才能进行正确的路由选择

链路状态算法需要各个路由器从网络代理获取路由信息表，在一个规模巨大的网络结构中，管理和处理信息需要高速的CPU处理能力和大量的内存

RIP可能出现的问题
  • 不一致的路由信息会导致路由环路，各种协议都应尽力避免路由环路。

10.4.0.0 down之后，B没有来得及更新路由信息，而C从B学到10.4.0.0的路由信息，从而形成环路，条数会不断上涨。

对跳数设置最大值以防止环路。

  • 解决办法：

水平分割-永远不会将一个路由信息从接收这个信息的接口上发送回去。

当一个网络不可达之后，把路由的距离设置为无限大，让所有路由器知道这个网络不可达。

当出现网络拓扑变化的时候，立即向外发布更新。

  • 总结 RIP防止路由环路技术

1.定义最大跳数

2.水平分割

3.触发更新

4.抑制计时器

5.路由毒化

OSPF协议

Open Shortest Path First，底层使用最短路径优先算法。

Link State数据结构

邻居表 Neighbor table

  • 邻接数据库
  • 保存所有已知邻居

拓扑表 Topology table

  • 又称为链路状态数据库(LSDB)

路由表 Routing table，由拓扑表运行 SPF算法得出

  • 又称为转发数据库(forwarding database/RIB)
  • 保存到达目标网络的最优路径
链路状态型路由协议与距离矢量型路由协议比较
  • 相比距离矢量型路由协议，链路状态型路由协议保存更多的、更详细的网络拓扑信息
  • 每个路由器保存全网拓扑信息
  • 一般来讲，链路状态协议比距离矢量型协议判断决策更加精确
链路状态协议存在的问题

每个路由器需要维护全网拓扑信息

  • 占用大量内存
  • 进行路由计算时占用大量CPU资源
  • 每次路由计算时间较长
  • 当细节网络出现变动时，全网路由器均需要重新收敛计算
OSPF分层网络架构
  • OSPF强制使用分层架构
  • OSPF定义了网络双层结构，定义了区域概念，通过区域划分，实现双层的网络架构
    • 传输区域 Transit area （骨干）
    • 普通区域 Regular areas（非骨干）

  • Backbone Area0维护全网信息
  • 普通Area区域维护各自的区域的信息
  • 不同区域之间通信，需要通过Backbon Area0转发
  • OSPF区域特点
    • 最小化路由条目数量
    • 本地拓扑变动
    • 将LSA泛洪限制在区域内
    • 需要一个分层的网络设计
  • OSPF设计原则
    • 网络内有且只有一个区域0
    • 普通区域通过区域0互联，普通区域之间不能有邻街关系
    • 每个区域的网络最好是连续的，可汇总的
    • ABR负责向区域0通告所在区域信息，最好可以汇总
OSPF计算

路由器通过对LSDB保存的信息使用 Dijkstra’s SPF算法进行计算获得到达目标网络的最优路由。

  • 每个路由器拥有所在区域的全部链路状态信息
  • 每个路由器将自己作为根，通过LSDB信息使用SPF算法构建SPF树
  • 通过运行SPF算法，比较不同路径的开销值，选择到达目标网络的最优路径，并删除环路
  • 计算出来的最优路径存入转发数据库，即路由表
多路访问网络

.svg)

  • MA网路，通过路由器的一个接口，不通过三层设备转发可以直接到达多台其他设备的网络叫做多路访问网络。
  • MA网络自身的特性会导致建立大量的邻接关系，LSA流量也会响应增大，为了解决这个问题，OSPF在MA网络中定义了DR/BDR的概念
  • 在MA网络中DR和BDR选举是必须的
  • 所有的路由器只与DR和BDR建立邻接关系
  • 发送给DR和BDR的数据包使用组播地址224.0.0.6
  • DR发送给其他路由器的数据包使用组播地址224.0.0.5
选举DR与BDR
  • Hello 数据包通过组播形式发送
  • 接口拥有最高OSPF优先级成为DR，拥有次高优先级的为BDR
  • 优先级相同时使用router-id进行比较，最高的成为DR
  • DR的选举是非抢占的

DR监听224.0.0.6

DR notifies others on 224.0.0.5

Hello数据包
  • 数据包内容
    • Router ID
    • Hello and dead intervals *
    • Neighbors (记录邻居关系)
    • Area ID*(这个接口所处的区域ID)
    • Router pripority (路由的优先级)
    • DR IP address (MA DR地址)
    • BDR IP address (MA BDR地址)
    • Authentication password * (认证的密码)
    • Stub area flag *
链路状态型路由协议的优点
  • 快速收敛Fast convergence：拓扑变化会立即被通告出去
  • 更强的路由环路防止：
    • 路由器知道全网拓扑
    • 链路状态数据包是顺序的和可靠传输的
  • 通过合理的网络设计，可以最优化路由器资源使用
链路状态协议
  • 对资源的使用
    • 内存（三张表，adjacency, topology, forwarding）
    • CPU（Dijkstra算法可能占用大量的CPU时间）
  • 需要严格的网络设计（当区域数量很多时）
  • 当出现区域断裂时恢出现严重问题
  • 配置较为复杂，尤其在WAN网络中
  • 排错较为复杂，很多时候需要分析LSDB
